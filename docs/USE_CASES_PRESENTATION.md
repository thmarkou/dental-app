# Use cases — δοκιμές & παρουσίαση

> **Σκοπός:** Κάθε σεναρίο εξηγείται για αρχάριο — *γιατί*, *τι κάνουμε*, *βήμα-βήμα*, *τι πρέπει να δεις*.  
> **Χρήση:** (1) έλεγχος ότι όλα δουλεύουν, (2) demo σε πελάτη/ομάδα.  
> **Εφαρμογή:** iPhone με **Release** build από Xcode (όχι Expo Go).  
> **Login προεπιλογής:** `admin` / `admin123` — άλλαξε κωδικό από Ρυθμίσεις πριν production.

---

## Πώς διαβάζεις κάθε use case

| Τμήμα | Τι σημαίνει |
|-------|-------------|
| **Σκοπός** | Γιατί υπάρχει αυτή η λειτουργία στο ιατρείο |
| **Προϋποθέσεις** | Τι πρέπει να ισχύει πριν ξεκινήσεις |
| **Βήματα** | Ακριβής σειρά πατημάτων |
| **Αναμενόμενο αποτέλεσμα** | Τι **πρέπει** να δεις — αν όχι, θεωρείται αποτυχία |
| **Για την παρουσίαση** | Μία πρόταση που μπορείς να πεις δυνατά |
| **Σημειώσεις** | Όρια, προαιρετικά, συχνά λάθη |

**Κατάσταση δοκιμής:** `[ ]` άδειο = δεν δοκιμάστηκε · `✅` = πέρασε · `❌` = απέτυχε (γράψε τι είδες).

---

## Χάρτης εφαρμογής (μετά το login)

Κάτω μπάρα (αριστερά → δεξιά):

| Tab | Τι κάνει |
|-----|----------|
| **Σήμερα** | Ροή ημέρας + επισκόπηση dashboard |
| **Ασθενείς** | Λίστα, στοιχεία, οδοντόγραμμα, λογιστήριο, τιμολόγια |
| **Ραντεβού** | Ημέρα / εβδομάδα / μήνας / έτος |
| **Κλινική** | Ίδια ροή ημέρας (άλλο tab για ροή κλινικής) |
| **Ταμείο** | Ημερήσια εισπράξεις ανά τρόπο πληρωμής |
| **Αναφορές** | KPIs μήνα, PDF/CSV, σύνδεση αποθήκης |
| **Ρυθμίσεις** | Ιατρείο, αποθήκη, υπενθυμίσεις, backup |

Από **ασθενή** ανοίγουν υπο-οθόνες: Οδοντόγραμμα, Ιστορικό, Λογιστήριο, Έγγραφα, Τιμολόγια & αποδείξεις, Σχέδια θεραπείας.

---

## Προετοιμασία πριν τις δοκιμές

1. iPhone: app εγκατεστημένη (Xcode Release Run).
2. (Προαιρετικό Mac) `npm run release:preflight` — πράσινο.
3. Άνοιξε app → login `admin` / `admin123`.
4. Για demo με «γεμάτα» δεδομένα: δημιούργησε **1 ασθενή**, **2 ραντεβού**, **1 θεραπεία**, **1 τιμολόγιο** (βλ. UC-10 «Πλήρης ροή»).

# Επίπεδο 1 — Σύστημα & πρόσβαση

## UC-01 · Σύνδεση (Login)

**Σκοπός**  
Μόνο εξουσιοδοτημένο προσωπικό βλέπει δεδομένα ασθενών (ασφάλεια).

**Προϋποθέσεις**  
- App ανοιχτή στην οθόνη login.  
- Υπάρχει χρήστης `admin` (δημιουργείται αυτόματα στην πρώτη εκκίνηση DB).

**Βήματα**  
1. Πεδίο χρήστη: πληκτρολόγησε `admin`.  
2. Πεδίο κωδικού: `admin123`.  
3. Πάτα **Σύνδεση**.

**Αναμενόμενο αποτέλεσμα**  
- [ ] Μπαίνεις στο κύριο UI (φαίνονται τα tabs κάτω).  
- [ ] **Δεν** εμφανίζεται μήνυμα λάθους.  
- [ ] Tab **Σήμερα** (ή πρώτο tab) φορτώνει χωρίς κενή οθόνη για πάντα.

**Για την παρουσίαση**  
«Η εφαρμογή είναι offline-first· τα δεδομένα μένουν στη συσκευή και η πρόσβαση προστατεύεται με login.»

**Σημειώσεις**  
Λάθος κωδικός → μήνυμα σφάλματος, μένεις στο login.

**Κατάσταση:** `[ ]`

---

## UC-02 · Λανθασμένος κωδικός

**Σκοπός**  
Επιβεβαίωση ότι δεν μπαίνει κανείς με τυχαία στοιχεία.

**Προϋποθέσεις**  
Οθόνη login.

**Βήματα**  
1. Χρήστης: `admin`, κωδικός: `λάθος`.  
2. **Σύνδεση**.

**Αναμενόμενο αποτέλεσμα**  
- [ ] Μήνυμα σφάλματος (όχι κενή οθόνη).  
- [ ] **Δεν** ανοίγουν τα tabs.

**Κατάσταση:** `[ ]`

---

## UC-03 · Αποσύνδεση (Logout)

**Σκοπός**  
Όταν φεύγει το προσωπικό από τη συσκευή, να μην μένει ανοιχτή η πρόσβαση.

**Προϋποθέσεις**  
Είσαι logged in.

**Βήματα**  
1. Tab **Ρυθμίσεις** (κάτω δεξιά).  
2. Κύλισε κάτω → **Αποσύνδεση** (κόκκινο).  
3. Επιβεβαίωσε αν ζητηθεί.

**Αναμενόμενο αποτέλεσμα**  
- [ ] Επιστροφή στην οθόνη login.  
- [ ] Πάτα πίσω στο hardware tab — **δεν** μπαίνεις στην app χωρίς login.

**Για την παρουσίαση**  
«Σε κοινό iPad στο ιατρείο, το logout κλείνει την πρόσβαση αμέσως.»

**Κατάσταση:** `[ ]`

---

## UC-04 · Εγγραφή νέου χρήστη (Sign up)

**Σκοπός**  
Δεύτερος χρήστης (π.χ. γραμματεία) μπορεί να δημιουργήσει λογαριασμό.

**Προϋποθέσεις**  
Logged out.

**Βήματα**  
1. Στην login, πάτα σύνδεσμο/κουμπί **Εγγραφή** (αν υπάρχει).  
2. Συμπλήρωσε username, password, επιβεβαίωση.  
3. Ολοκλήρωσε εγγραφή → login με τα νέα στοιχεία.

**Αναμενόμενο αποτέλεσμα**  
- [ ] Επιτυχής εγγραφή και login.  
- [ ] Ίδιο κύριο UI με admin.

**Σημειώσεις**  
Αν δεν υπάρχει κουμπί Sign up στο UI, σημείωσε N/A — μόνο admin από migration.

**Κατάσταση:** `[ ]`

---

## UC-05 · Αλλαγή κωδικού

**Σκοπός**  
Production ασφάλεια — αντικατάσταση `admin123`.

**Προϋποθέσεις**  
Logged in ως admin.

**Βήματα**  
1. **Ρυθμίσεις** → ενότητα κωδικού (άνοιξε φόρμα αν είναι κλειστή).  
2. Τρέχων κωδικός: `admin123`.  
3. Νέος + επιβεβαίωση (π.χ. `Admin2026!Secure`).  
4. Αποθήκευση.  
5. **Αποσύνδεση** → login με **νέο** κωδικό.

**Αναμενόμενο αποτέλεσμα**  
- [ ] Μήνυμα επιτυχίας.  
- [ ] Παλιός κωδικός **δεν** δουλεύει.  
- [ ] Νέος κωδικός δουλεύει.

**Σημειώσεις**  
Για demo μπορείς να το παραλείψεις — κράτα `admin123` σε test device.

**Κατάσταση:** `[ ]`

- [ ] Κατάσταση (π.χ. προγραμματισμένο / επιβεβαιωμένο) φαίνεται με χρώμα ή κείμενο.

# Επίπεδο 4 — Ραντεβού

**Σκοπός**  
Προγραμματισμός επίσκεψης.

**Προϋποθέσεις**  
Τουλάχιστον 1 ασθενής.

**Βήματα**

**Για την παρουσίαση**  
«Πλάνο εβδομάδας και μήνας στα ελληνικά — βλέπουμε αμέσως ποιος έρχεται.»

## UC-32 · Πλέγμα μήνα (2+ ραντεβού ίδια μέρα)

Έλεγχος ανάγνωσης όταν έχεις **πολλά** ραντεβού την ίδια μέρα.

3. **Συντομογραφία** → ξαναδες μήνα.

**Αναμενόμενο αποτέλεσμα**  
- [ ] **Πλήρες:** ώρα γραμμή 1, όνομα γραμμή 2 (ανά ραντεβού).  
- [ ] **Συντομογραφία:** μία γραμμή `09:30 Μ. Επώνυμο`.  
- [ ] Αν >3 ραντεβού: `+N ακόμα` στα ελληνικά.

**Κατάσταση:** `[ ]`
