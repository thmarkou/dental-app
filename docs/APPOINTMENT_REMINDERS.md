# Υπενθυμίσεις ραντεβού

## Τοπικές ειδοποιήσεις (Push στο τηλέφωνο)

- Προγραμματίζονται με `expo-notifications` όταν δημιουργείται/ενημερώνεται ραντεβού.
- **Δεν** χρειάζονται διακομιστής· συνήθως λειτουργούν και με κλειστή εφαρμογή.
- Μετά επανεκκίνηση συσκευής η συμπεριφορά εξαρτάται από iOS/Android — δεν εγγυάται το 100%.

## SMS

- Η αποστολή γίνεται μέσω HTTP gateway (`FEATURE_SMS_REMINDERS` + `SMS_GATEWAY_API_KEY`).
- Η εφαρμογή καταγράφει `scheduled_for` στη βάση και στέλνει όταν:
  1. Ανοίγει η εφαρμογή (αρχικοποίηση), ή
  2. Επιστρέφει στο προσκήνιο (`AppState` → `active`) — `reminderTick.service.ts`.

### Περιορισμός MVP (προσωρινό)

**Σήμερα:** το SMS στέλνεται όταν ο χρήστης **ξανανοίγει** την εφαρμογή — δηλαδή μετά από κλείσιμο ή όταν επανέρχεται από παρασκήνιο (όχι ενώ τρέχει στο background χωρίς interaction).

**Μελλοντικά (μαζί με myDATA / cloud):** αποστολή ακριβώς 24 ώρες πριν **χωρίς** ανοιχτή εφαρμογή, μέσω:

- Server cron που καλεί το SMS gateway, ή
- Third-party scheduler.

Αυτό δεν είναι μέρος της τρέχουσας φάσης· θα σχεδιαστεί στο τέλος του πλάνου.

## Remote push (FCM / Expo Push)

- `FEATURE_REMOTE_PUSH=true` και έγκυρο `extra.eas.projectId` στο `app.json`.
- Η εφαρμογή αποθηκεύει το Expo push token τοπικά (φάση 4.2a).
- Η **αποστολή** υπενθυμίσεων από cloud (4.2b) δεν υλοποιείται στο MVP — χρειάζεται backend ή κλήση στο [Expo Push API](https://docs.expo.dev/push-notifications/sending-notifications/).
